# wrappedether.org v2.0 - Animated Explainer Plan

**Status:** Planning Phase
**Target:** Modern, animated single-page explainer for WETC
**Inspiration:** weth.io (archived), ethereum.org, uniswap.org
**Timeline:** TBD (public good, no deadline pressure)

---

## Executive Summary

Transform wrappedether.org from minimal HTML site into a modern, animated explainer page that educates users about WETC in 60 seconds and routes them to wrap/unwrap interfaces.

**Core Principle:** Visual education over text walls. Make wrapping understandable through design.

---

## Design Philosophy

### Inspiration from weth.io

While weth.io is now offline, it was the gold standard for wrapped token education during the DeFi boom. Key lessons:

1. **Visual First** - Diagrams explain wrapping better than paragraphs
2. **Progressive Disclosure** - Start simple (what?), then deeper (why?, how?)
3. **Minimal Friction** - Single page, clear CTAs, no navigation complexity
4. **Technical Accuracy** - Contract addresses prominent, verification links clear
5. **Design = Legitimacy** - Professional design builds trust (critical for DeFi)

### Our Approach for WETC

**Target Audience:**
- **Primary:** ETC holders who want to use DeFi (ETCswap, ClassicUSD)
- **Secondary:** Developers integrating WETC into protocols
- **Tertiary:** General crypto users learning about ETC

**User Journey:**
1. Land on site → "What is WETC?" (15 seconds)
2. Understand why needed → "ETC predates ERC-20" (20 seconds)
3. Learn how it works → "1:1 trustless wrapping" (15 seconds)
4. Know where to wrap → "ETCswap or Classic OS" (10 seconds)
5. **Total: 60 seconds to educated user**

### 2. Timeline - Scroll-Triggered Animation

**Effect:** Timeline items fade in and connect as user scrolls

```typescript
const timelineItems = [
  { year: '2015', title: 'ETC Launches', icon: ETCIcon },
  { year: '2017', title: 'ERC-20 Standard', icon: DocumentIcon },
  { year: '2020+', title: 'DeFi Boom', icon: DeFiIcon }

<motion.div className="relative">
  {/* Connecting line */}
  <motion.div
    className="absolute top-0 left-1/2 h-full w-1 bg-etc-green"
    initial={{ scaleY: 0 }}
    whileInView={{ scaleY: 1 }}
    viewport={{ once: true, margin: '-100px' }}
    transition={{ duration: 1.5 }}
  />

  {/* Timeline items */}
  {timelineItems.map((item, i) => (
    <motion.div
      key={item.year}
      initial={{ opacity: 0, x: i % 2 === 0 ? -50 : 50 }}
      whileInView={{ opacity: 1, x: 0 }}
      viewport={{ once: true, margin: '-100px' }}
      transition={{ delay: i * 0.3, duration: 0.6 }}
    >
      {/* Item content */}
    </motion.div>
  ))}
</motion.div>
```

```typescript
const [mode, setMode] = useState<'wrap' | 'unwrap'>('wrap')

      className={mode === 'wrap' ? 'active' : ''}

    </button>
    <button
      onClick={() => setMode('unwrap')}
      className={mode === 'unwrap' ? 'active' : ''}
    >
      Unwrap
    </button>
  </div>

  {/* Diagram */}
  <motion.div
    key={mode}
    initial={{ opacity: 0, y: 20 }}
    animate={{ opacity: 1, y: 0 }}
    exit={{ opacity: 0, y: -20 }}
    transition={{ duration: 0.4 }}
  >
    {mode === 'wrap' ? <WrapFlow /> : <UnwrapFlow />}
  </motion.div>
</div>
```

```typescript
const [copied, setCopied] = useState(false)

const copyAddress = async (address: string) => {

<motion.button

        exit={{ opacity: 0, y: -10 }}
```

## Migration Strategy

### From v1.0 (GitHub Pages) to v2.0 (Vercel)

**Step 1: Build v2.0 (no disruption to v1.0)**

- Deploy to Vercel staging URL (wrappedether.vercel.app)
- Test thoroughly

**Step 2: DNS Cutover**
- Update CNAME record: www.wrappedether.org → Vercel
- Verify SSL certificate issued
- Monitor for DNS propagation (24-48 hours)

**Step 3: Archive v1.0**
- Create git tag: `v1.0-final`
- Archive v1.0 HTML in `/v1-archive` branch
- Keep GitHub Pages deployment as fallback

**Step 4: Monitor & Iterate**
