## Animation Strategy

### WETH.io Reference → WETC Custom

Each WETH.io GIF will be recreated as a custom SVG + Framer Motion component:

| # | WETH.io GIF | Concept | WETC Component | Priority |
|---|-------------|---------|----------------|----------|
| 1 | WETH_01.gif | Blockchain + dApp boxes | `<BlockchainDapp />` | High |
| 2 | WETH_02.gif | ERC-20 jar (token can't enter) | `<Erc20Jar />` | High |
| 3 | WETH_03.gif | Smart contract vacuum | `<SmartContract />` | Medium |
| 4 | WETH_04.gif | Wrap/unwrap conveyor | `<WrapMachine />` | High |
| 5 | WETH_05.gif | Evolution (box opening) | `<Evolution />` | Low |

### Animation Generation Approach

**Primary Method: Claude SVG + Framer Motion**

```
1. Analyze WETH.io GIF → Identify key frames and motion
2. Prompt Claude → Generate isometric SVG with grouped elements
3. Add Framer Motion → Animate paths, transforms, opacity
4. Iterate → Refine timing, easing, colors
```

**SVG Style Guide:**
- Isometric 3D perspective (same as WETH.io)
- ETC color palette: Green (#3AB83A), Black (#0a0a0a), White (#fff)
- Clean line art with diagonal hatching patterns
- Grouped elements for animation (`<g id="coin">`, `<g id="machine">`)

### Framer Motion Patterns

```tsx
// Scroll-triggered animation
<motion.div
  initial={{ opacity: 0, y: 50 }}
  whileInView={{ opacity: 1, y: 0 }}
  viewport={{ once: true }}
  transition={{ duration: 0.6 }}
>

// Looping animation
<motion.path
  animate={{ rotate: 360 }}
  transition={{ duration: 3, repeat: Infinity, ease: 'linear' }}
/>

// Staggered children
<motion.div variants={container} initial="hidden" animate="show">
  {items.map(item => (
    <motion.div variants={item} key={item.id} />
  ))}
</motion.div>
```

## Animation Generation Tools

### Recommended: Claude SVG + Framer Motion

**Pros:**
- Full control over style and colors
- Lightweight (SVG + JS)
- Easy to iterate
- Matches WETH.io aesthetic perfectly

**Workflow:**
1. Study WETH.io GIF frame-by-frame
2. Describe animation to Claude with style guide
3. Claude generates SVG with animation-ready structure
4. Add Framer Motion for movement
5. Test and refine

### Alternative Tools

| Tool | Type | When to Use |
|------|------|-------------|
| **Rive** | Interactive | Complex state-based animations |
| **Lottie** | JSON export | If designer provides After Effects files |
| **SVGator** | Online editor | Quick prototyping |
| **Figma** | Design tool | Initial SVG asset design |

### AI Animation Tools (Experimental)

| Tool | Quality | Control | Cost |
|------|---------|---------|------|
| Haiper AI | Medium | Low | Free |
| Pika Labs | Medium | Low | Free |
| RunwayML Gen-2 | High | Low | Paid |
| Leonardo.ai | Medium | Medium | Free tier |

**Note:** AI video tools lack the precise control needed for our isometric style. Claude SVG generation + Framer Motion is the recommended approach.

## Migration Path

### From v0.1 to v0.2

1. **Keep v0.1 live** - Current static site stays at www.wrappedether.org
2. **Build v0.2 in branch** - Develop in `v0.2` or `next` branch
3. **Deploy to Vercel staging** - Test at `wrappedether-v02.vercel.app`
4. **DNS cutover** - Point www.wrappedether.org to Vercel
5. **Archive v0.1** - Keep in `v0.1` branch for history

### Rollback Plan

If v0.2 has issues:
1. Revert DNS to GitHub Pages
2. v0.1 static site is immediately live
3. Fix issues in v0.2
4. Re-deploy when ready
